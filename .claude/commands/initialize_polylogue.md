# Initialize Polylogue

Deploy all commands and subagents from `configs/` to `.claude/` so they are available as slash commands and subagents in Claude Code.

---

## When to Run

Run this command after:
- Cloning the repository
- Creating or modifying any command in `configs/`
- Creating or modifying any subagent in `configs/`
- Pulling changes that affect `configs/`

---

## Execution

### Step 1: Create Target Directories

Ensure these directories exist:
- `.claude/commands/`
- `.claude/agents/scenario/`
- `.claude/agents/agent/`
- `.claude/agents/presentation/`
- `.claude/agents/discussion/`
- `.claude/agents/evaluation/`
- `.claude/agents/personas/`

### Step 2: Sync Commands

Copy all command files from their layer directories to `.claude/commands/`:

| Source | Target |
|--------|--------|
| `configs/scenario/commands/create_scenario.md` | `.claude/commands/create_scenario.md` |
| `configs/agent/commands/generate_profiles.md` | `.claude/commands/generate_profiles.md` |
| `configs/agent/commands/generate_personas.md` | `.claude/commands/generate_personas.md` |
| `configs/agent/commands/validate_profile.md` | `.claude/commands/validate_profile.md` |
| `configs/presentation/commands/generate_presentation.md` | `.claude/commands/generate_presentation.md` |
| `configs/discussion/commands/begin_discussion.md` | `.claude/commands/begin_discussion.md` |
| `configs/discussion/commands/continue_discussion.md` | `.claude/commands/continue_discussion.md` |
| `configs/evaluation/commands/evaluate_presentation.md` | `.claude/commands/evaluate_presentation.md` |
| `configs/evaluation/commands/evaluate_discussion.md` | `.claude/commands/evaluate_discussion.md` |
| `configs/system/commands/initialize_polylogue.md` | `.claude/commands/initialize_polylogue.md` |

Skip any source files that don't exist yet (not all layers may be built).

### Step 3: Sync Subagents

Copy all subagent files from their layer directories:

| Source | Target |
|--------|--------|
| `configs/scenario/subagents/scenario-generator.md` | `.claude/agents/scenario/scenario-generator.md` |
| `configs/agent/subagents/profile-generator.md` | `.claude/agents/agent/profile-generator.md` |
| `configs/agent/subagents/persona-generator.md` | `.claude/agents/agent/persona-generator.md` |
| `configs/presentation/subagents/section-generator.md` | `.claude/agents/presentation/section-generator.md` |
| `configs/discussion/subagents/speaker-selector.md` | `.claude/agents/discussion/speaker-selector.md` |
| `configs/discussion/subagents/stage-tracker.md` | `.claude/agents/discussion/stage-tracker.md` |
| `configs/evaluation/subagents/evaluator.md` | `.claude/agents/evaluation/evaluator.md` |

Skip any source files that don't exist yet.

### Step 4: Verify Persona Directories

List any existing persona directories in `.claude/agents/personas/`. These are generated by `/generate_personas` and should not be overwritten by initialization.

Report their existence but don't modify them.

### Step 5: Report

```
Polylogue initialized.

Commands synced:
  ├── {list of commands synced}
  └── {count} commands

Subagents synced:
  ├── {list of subagents synced}
  └── {count} subagents

Personas found:
  ├── {list of scenario_id directories in .claude/agents/personas/}
  └── {count} scenario persona sets

Skipped (not yet built):
  └── {list of source files that don't exist}
```

---

## Notes

- This command is idempotent — safe to run multiple times.
- It overwrites target files with current source files. This is intentional: `configs/` is the canonical source.
- Persona files (`.claude/agents/personas/`) are NOT synced by this command — they are generated by `/generate_personas`.
- If a command or subagent is deleted from `configs/`, the corresponding file in `.claude/` will NOT be automatically removed. Delete manually if needed.
